### Параметры конфигурации postgresql.conf
#### Основной файл конфигураций
Главный файл - это postgresql.conf 
Файл находится по пути указанному в переменной PGDATA
Ubuntu - /etc/postgresql/16/main
Если один и тот же параметр указан в конфигурационных файлах несколько раз, будет использоваться значение, считанное последним.
#### При изменении необходимо перечитать файл
```bash
pg_ctl reload
pg_ctlcluster 16 main reload
```

Изменение ряда параметров требуют перезапуска службы postgresql.
shared_buffers and work mem
```sql
=> SELECT pg_reload_conf();
=> SHOW config_file;
```
#### Подключение файлов конфигураци
* include_dir - каталог с дополнительными файлами
* include - дополнительный файл конфигурации
* include_if_exists - дополнительный файл конфигурации, если он существует
Просмотр информации
```bash
grep -A3 ^include /etc/postgresql/16/main/postgresql.conf

include_dir = 'conf.d'                  # include files ending in '.conf' from
                                       # a directory, e.g., 'conf.d'
#include_if_exists = '...'              # include file only if it exists
#include = '...'                        # include file
```
Всё что находится в директории conf.d будет подключено к файлу postgresql.conf

#### Представление pg_settings
```sql
=> SELECT name, setting, unit, boot_val, reset_val,
  source, sourcefile, sourceline, pending_restart, context
FROM pg_settings
WHERE name = 'work_mem'\gx

-[ RECORD 1 ]---+-------------------------------------------------
name            | work_mem
setting         | 16384
unit            | kB
boot_val        | 4096
reset_val       | 16384
source          | configuration file
sourcefile      | /var/lib/postgresql/16/main/postgresql.auto.conf
sourceline      | 3
pending_restart | f
context         | user

=> SELECT name, setting, unit, boot_val, reset_val,
  source, sourcefile, sourceline, pending_restart, context
FROM pg_settings
WHERE name = 'shared_buffers'\gx

-[ RECORD 2 ]---+----------------------------------------
name            | shared_buffers
setting         | 16384
unit            | 8kB
boot_val        | 16384
reset_val       | 16384
source          | configuration file
sourcefile      | /etc/postgresql/16/main/postgresql.conf
sourceline      | 130
pending_restart | f
context         | postmaster

```
Ключевые столбцы:
- name, setting, unit — название, значение и единица измерения параметра;
- boot_val — значение по умолчанию;
- reset_val — значение в начале сеанса;
- source — источник текущего значения параметра;
- sourcefile, sourceline — файл конфигурации и номер строки, если текущее значение было задано в файле;
- pending_restart — true, если значение изменено в файле конфигурации, но для применения требуется перезапуск сервера.
Столбец context:
- internal — изменить нельзя, значение задано при установке;
- postmaster — требуется перезапуск сервера;
- sighup — требуется перечитать файлы конфигурации,
- superuser — суперпользователь может изменить для своего сеанса;
- user — любой пользователь может изменить для своего сеанса.
```sql
SELECT sourcefile, sourceline, name, setting, applied 
FROM pg_file_settings 
WHERE sourcefile LIKE '%work_mem.conf' \gx

-[ RECORD 1 ]--------------------------------------------
sourcefile | /etc/postgresql/16/main/conf.d/work_mem.conf
sourceline | 1
name       | work_mem
setting    | 12MB
applied    | f
-[ RECORD 2 ]--------------------------------------------
sourcefile | /etc/postgresql/16/main/conf.d/work_mem.conf
sourceline | 2
name       | work_mem
setting    | 8MB
applied    | t

SHOW work_mem;
```
applied - f для первой строки показывает, что она не будет применена. 

### Файл postgresql.auto.conf
Файл конфигурации управляемый командами SQL.
```sql
ALTER SYSTEM добавляет или изменяет строку
SET параметр TO значение;
ALTER SYSTEM RESET параметр; удаляет строку
ALTER SYSTEM RESET ALL; удаляет все строки
```
Последним считывается файл postgresql.auto.conf. Он всегда располагается в каталоге данных. 
https://postgrespro.ru/docs/postgresql/16/sql-altersystem
```sql
=> ALTER SYSTEM SET work_mem TO '16MB';

=> SELECT pg_read_file('postgresql.auto.conf')
\g (tuples_only=on format=unaligned)

# Do not edit this file manually!
# It will be overwritten by the ALTER SYSTEM command.
work_mem = '16MB'

select pg_reload_conf(); 
# After change settings we should reload
```
### Управление параметрами на уровнях экземпляра и сеанса
Для изменения параметров во время сеанса можно использовать команду SET:

```sql
=> SET work_mem TO '24MB';
=> RESET work_mem;
```

Или функцию set_config:

```sql
=> SELECT set_config('work_mem', '32MB', false);
```

Третий параметр функции говорит о том, нужно ли устанавливать значение только для текущей транзакции (true) или до конца работы сеанса (false). Это важно при работе приложения через пул соединений, когда в одном сеансе могут выполняться транзакции разных пользователей.

#### Варианты вывода информации о параметре
```sql
SHOW work_mem;

\dconfig work_mem

SELECT current_setting('work_mem');

SELECT name, setting, unit
FROM pg_settings
WHERE name = 'work_mem'\gx
```

Доступна установка параметров внутри транзакции
```sql
BEGIN;

SET work_mem TO '64MB'; 

ROLLBACK;

SET LOCAL work_mem TO '64MB'; #до конца транзакции

set_config('work_mem','64MB',true); #до конца транзакции

COMMIT;
```
