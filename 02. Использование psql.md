### Работа в psql
#### Команды psql для переключения режима выравнивания:
- \a — переключает режим с выравниванием и без выравнивания;
- \t — переключает отображения строки заголовка и итоговой строки.
- \pset — вывод параметров
- \pset fieldsep ' ' — переопределение разделителя
- \x — расширенный формат удобно включать, когда в таблице большое количество столбцов
```sql
SELECT * FROM pg_tables WHERE tablename = 'pg_proc' \gx
```
Применение расширенного режима к конкретному запросу
```sql
SELECT * FROM pg_tables WHERE tablename = 'pg_proc';
```
При выполнении SQL запроса важно его завершать знаком ;

#### Выполнение скриптов
Еще один вариант отправить запрос на выполнение — команда \g
Благодаря ей можно применять специальные параметры к конкретному запросу

Пронумеровать строки вывода sql запроса
```sql
SELECT format('SELECT count(*) FROM %I;', tablename)
FROM pg_tables
LIMIT 3
\g (tuples_only=on format=unaligned) | cat -n
```

Записать вывод запроса в файл
```sql
SELECT format('SELECT count(*) FROM %I;', tablename)
FROM pg_tables
LIMIT 3
\g (tuples_only=on format=unaligned) tmp/dba1_log
```

3 варианта выполнение скрипта dba1_log из каталога tmp 
```
\i tmp/dba1_log
psql < filename
psql -f filename
```

Выполнения запроса и подзапросов
```sql
SELECT format('SELECT count(*) FROM %I;', tablename)
FROM pg_tables
LIMIT 3
\gexec
```
\gexec - execute query, then execute each value in its result
Выполнение основного запроса и тех, что у нас получились.
#### Переменные psql и управляющие конструкции
```sql
\getenv User USER
# Запомним в переменной psql User значение переменной окружения USER

\set Test Hi
# Запомним значение в переменную

\set
# Вывод всех переменных в консоль

\echo :Test :User!
# Вывод переменых в консоль

\unset Test
# Сбросить переменную

SELECT now() AS curr_time \gset
# Запись результата запроса в переменную curr_time
```
Условные операторы
```sql
\echo :{?working_dir}
# Проверяет существует ли переменная

\if :{?working_dir}
   -- переменная определена
\else
   -- в качестве значения можно установить результат выполнения команды ОС
   \set working_dir `pwd`
\endif
# Пример скрипта. Проверка переменной и запись значения по умолчанию
```

### Настройка psql
При запуске psql запускаются 2 скрипта (при наличии)
- сначала общий системный скрипт psqlrc;
- затем пользовательский файл .psqlrc.
```
pg_config --sysconfdir
# Каталог расположения системного скрипта
```

```sql
\set top5 'SELECT tablename, pg_total_relation_size(schemaname||''.''||tablename) AS bytes FROM pg_tables ORDER BY bytes DESC LIMIT 5;'
```
Запись текста запроса в переменную
Данную команду можно поместить в файл .psqlrc и тогда переменная :top5 будет сразу же доступна через консоль.

Изменение приглашения psql
https://postgrespro.ru/docs/postgresql/16/app-psql#APP-PSQL-PROMPTING
Добавить в файл .psqlrc следующую запись
```bash
\set PROMPT1 '%n@%/%R%x%# '
\set PROMPT2 '%n@%/%R%x%# '
```
%n - Добавляет имя пользователя подключенного к БД
@ - Используется, как разделитель

https://postgrespro.ru/docs/postgresql/16/sql-savepoint
```bash
\set ON_ERROR_ROLLBACK on
```
Позволяет продолжить транзакцию несмотря на ошибки.
Теперь перед каждой командой транзакции неявно будет устанавливаться точка сохранения, а в случае ошибки будет происходить автоматический откат к этой точке. Это даст возможность продолжить выполнение команд транзакции.
