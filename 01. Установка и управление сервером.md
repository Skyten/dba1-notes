### Основные понятия
Один экземпляр PostgreSQL одновременно работает с несколькими
базами данных. Этот набор баз данных называется кластером баз
данных.

#### Создание файла конфигурации
>[!warning] Важно
>Если сборка производится повторно, например с другими параметрами конфигурации, то предварительно нужно убрать файлы, оставшиеся от предыдущей сборки:
>```bash
>student$ make distclean
>```

```bash
student$ cd /home/student/postgresql-16.0
student$ ./configure --prefix=/home/student/pgsql16 --with-pgport=5555
```

У скрипта configure есть большое множество параметров для запуска
 --prefix — каталог установки, по умолчанию /usr/local/pgsql;
 --enable-debug — для включения отладочной информации.
 --with-pgport — можно указать нестандартный порт отличный от 5432

#### Сборка и установка PostgreSQL
Возможные варианты сборки:
1. make — сборка только сервера;
2. make world — сборка сервера, всех расширений и документации.

Установка всё и сразу 
```bash
make
make install-world (сборка должна быть выполнена командой make world)
```

Сборка сервера
```bash
student$ make
```

Установка сервера
```bash
student$ make install
```

Установка расширений
Важно выполнить следующие действия в каталоге contrib
```bash
student$ cd contrib
student$ make
student$ make install
```

#### Создание кластера

>[!warning] Важно
>Новый кластер баз данных не может принадлежать суперпользователю ОС 
>PGDATA — удобная переменная для каталога кластера 
>Файлы конфигурации создаются в  каталоге PGDATA
>Желательно включить расчет контрольных сумм для страниц с данными

```bash
mkdir /home/student/pgsql16/data
export PGDATA=/home/student/pgsql16/data
# PGDATA - переменная для расположения БД
export PATH=/home/student/pgsql16/bin:$PATH
# Добавлениие утилит postgresql в переменную PATH
initdb -U postgres -k -D /home/student/pgsql16/data
```
- В ключе -U можно указать имя суперпользователя СУБД. Если его не задать, то используется имя пользователя ОС, в нашем случае student. Обычно таким пользователем является postgres, так что зададим это имя явно.
- Ключ -k, или --data-checksums включает подсчет контрольных сумм страниц, что позволяет своевременно обнаруживать повреждения данных.
- Если переменная PGDATA не установлена, то в ключе -D следует указать каталог для данных. В нашем случае этот ключ можно было бы не указывать.
### Управление сервером
#### Запуск сервера
```bash
pg_ctl start -l /home/student/logfile
# - В ключе -l укажем файл журнала сообщений сервера.
# - Ключ -D опускаем, поскольку задана переменная PGDATA.
pg_ctl stop
pg_ctl restart
pg_ctl reload
# Загрузка новой конфигурации без перезагрузки
```
#### Проверка того, что БД запустилась 
```bash
psql -U postgres -p 5555 -c 'SELECT now();'
```
Выполнение запроса на вывод текущего времени
### Пакетная установка
```bash
sudo apt install -y postgresql-16
sudo ls -l /usr/lib/postgresql/16
# Каталог установки
/usr/lib/postgresql/16/bin/pg_config --configure
# Просмотр конфигурации postgresql
ls -l /var/lib/postgresql/16/main
# Кластер по-умолчанию называется main
```
Для настройки собственного кластера используется команда pg_createcluster
### Управление сервером в Ubuntu
Для управления сервером в ОС Ubuntu (Debian) используется утилита pg_ctlcluster
Утилита pg_ctlcluster позволяет управлять несколькими серверами разных версий. В качестве параметров ей передаются:
- 16 — номер версии сервера;
- main — название сервера;
- команда
```bash
pg_ctlcluster 16 main stop
pg_lsclusters 16 main
# Просмотр состояния кластера
pg_ctlcluster 16 main start
pg_ctlcluster 16 main restart
pg_ctlcluster 16 main status
# Получение статуса кластера
pg_ctlcluster 16 main reload
# Перечитать файлы конфигурации
```
Журнал сообщений сервера
```bash
ls -l /var/log/postgresql/postgresql-16-main.log
```
#### Включение расчёта контрольных сумм кластера
https://postgrespro.ru/docs/postgresql/16/app-pgchecksums?lang=ru-en
```bash
pg_ctlcluster 16 main stop
/usr/lib/postgresql/16/bin/pg_checksums --check -D /var/lib/postgresql/16/main
# Проверка включены ли контрольные суммы
/usr/lib/postgresql/16/bin/pg_checksums --enable -D /var/lib/postgresql/16/main
# Включение контрольных сумм
pg_ctlcluster 16 main start
```
